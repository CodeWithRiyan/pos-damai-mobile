## Commands

```bash
# Install dependencies (uses pnpm)
pnpm install

# Start dev server (default: dev API)
pnpm start

# Start with specific environment
pnpm start:local     # http://localhost:7071
pnpm start:dev       # api-dev-pos-damai.riyansolusi.com
pnpm start:sit       # api-sit-pos-damai.riyansolusi.com
pnpm start:demo      # api-demo-pos-damai.riyansolusi.com
pnpm start:prod      # api-pos-damai.riyansolusi.com

# Run on device/emulator
pnpm android
pnpm ios

# Lint
pnpm lint

# Type check + export (build validation)
pnpm build:check

# Generate DB migrations after schema changes
pnpm generate
```

**Path alias**: `@` maps to the project root (e.g., `@/db`, `@/stores/auth`).

## Architecture

### Overview

POS Damai is a React Native (Expo) point-of-sale app that works **offline-first** using a local SQLite database. Data syncs with a REST API server. The app is landscape-only with a sidebar navigation pattern.

### Routing (Expo Router, file-based)

- `app/_layout.tsx` — root layout; handles DB init, auth rehydration, and auth redirects
- `app/login.tsx` — login screen
- `app/(main)/` — authenticated section containing all POS features:
  - `transaction/` — POS checkout and receipts
  - `purchasing/` — purchase orders
  - `shift/` — cashier shift management
  - `management/` — products, categories, brands, customers, users, etc.
  - `report/` — sales, purchasing, stock, return, customer reports
  - `finance/` — financial receipts/payments
  - `sync/` — manual sync control UI
  - `setting/` — app settings

### State Management (Zustand)

All stores are in `stores/`. Key stores:

- `auth.ts` — user profile and JWT; uses `authStorageAdapter` for persistence
- `transaction.ts` — shopping cart and active transaction
- `sync-queue-store.ts` — pending local operations awaiting sync
- `network-store.ts` — real-time internet connectivity status
- `shift.ts`, `purchasing.ts`, etc. — domain-specific transient state

### Local Database (Drizzle ORM + expo-sqlite)

- Schema defined in `lib/db/schema.ts`
- DB instance exported from `lib/db/index.ts` as `db`
- Migrations in `drizzle/` (generated by `pnpm generate`, applied at startup)
- Every synced table has a `_dirty: boolean` column — set to `true` when locally modified and reset after successful push
- `resetDb()` in `lib/db/index.ts` drops all tables and clears AsyncStorage; DB auto-reset on version change is **disabled** to protect unsync'd data

### Sync Engine (`lib/sync/sync-engine.ts`)

Two-phase sync:

1. **Pull**: Calls `/sync/bootstrap` (first run) or `/sync/incremental?since=<lastSyncAt>` — server wins, overwrites local master data
2. **Push**: Sends all records with `_dirty: true` to the server

`lastSyncAt` is stored in AsyncStorage. Deleting it forces a full bootstrap sync. Log prefix: `[Sync]`.

### API Layer (`lib/api/`)

- `client.ts` — Axios instance; reads `EXPO_PUBLIC_API_URL` (falls back to dev); JWT injected via request interceptor; auto-refreshes token on 401
- One file per domain (e.g., `products.ts`, `transactions.ts`) containing typed API functions

### Storage (`lib/storage.ts`)

Synchronous in-memory cache backed by AsyncStorage. Must call `initializeStorage()` before any reads. Two separate caches: `storageAdapter` (general) and `authStorageAdapter` (tokens).

### UI Stack

- **NativeWind v4** (Tailwind CSS for React Native) — use `className` props
- **Gluestack UI** — component library; provider in root layout
- **Lucide React Native** + **Solar Icons** — icons
- **React Hook Form** + **Zod** — forms and validation

### Permissions (`hooks/use-permission.ts`)

Role-based permission check via `usePermission()`. Permissions come from `profile.roles[].permissions[]`. Wildcard `*:*` grants super admin access. Use `PermissionGuard` component (`components/permission-guard.tsx`) to conditionally render UI.

### Adding DB Schema Changes

1. Modify `lib/db/schema.ts`
2. Run `pnpm generate` to create a new migration file in `drizzle/`
3. Migrations run automatically on next app startup via `initializeDb()`
